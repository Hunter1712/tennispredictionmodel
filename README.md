# Tennis Match Predictor

XGBoost model predicting ATP tennis match winners, with a web UI for browsing predictions.

## Quick Start

```bash
# Train model
python main.py

# Export predictions for web UI
python export_predictions.py
```

Then open `index.html` in a browser.

## Project Structure

```
├── main.py              # Entry point - trains model
├── pipeline.py          # Training pipeline orchestration
├── features.py          # Feature engineering
├── cleaner.py           # Data cleaning
├── loader.py            # CSV data loading
├── model.py             # Model training/evaluation
├── config.py            # Configuration
├── export_predictions.py # Generate predictions.js for web
├── index.html           # Web UI (uses predictions.js)
├── predictions.js       # Generated by export_predictions.py
├── data/                # CSV match data (1991-2026)
└── tennis_model.pkl     # Trained model
```

## How It Works

1. **Data**: Loads CSV files from `data/` folder (yearly files like `2024.csv`)
2. **Cleaning**: Removes incomplete matches, converts dates, handles missing values
3. **Features**: Builds player stats (win rate, recent form, surface skill) and creates diff features
4. **Training**: XGBoost classifier trained on 1991-2024, tested on 2025-2026
5. **Export**: Pre-computes all matchup predictions to JSON for web UI

## Web UI

- **Predict**: Select two players to see win probability
- **Results**: Model accuracy on test data
- **Rankings**: All players sorted by rank points

## Requirements

- Python 3.14+
- xgboost, pandas, scikit-learn, numpy
